# 酒店管理系统操作手册

本文档整理本项目的业务功能，说明各业务模块的用途与常用操作步骤，便于快速上手和演示。

## 系统概览
- 技术形态：纯 JavaScript 单页应用，所有数据存储在浏览器的 `localStorage` 中，无需后端依赖。
- 默认登录密码：`h123`（所有用户统一密码）。
- 导航模块：仪表盘、人员管理、房间管理、房间打扫、布草管理、资产管理、用户管理、系统管理。

## 登录
1. 打开页面后进入登录面板，选择一个已有账号（默认包含“管理员”“普通用户”）。
2. 输入默认密码 `h123` 并点击“立即登录”进入系统。
3. 右上角“退出登录”可切换账号。

## 业务模块与操作
### 仪表盘
- 查看房间总数、可用房间、入住中、维修/打扫中的实时统计。
- “房间概览”卡片展示部分房间，可直接办理入住、退房、指派打扫、结束维修等操作。
- “运行情况”展示员工数量、资产记录、布草件数、历史订单数量。

```mermaid
usecaseDiagram
actor Admin as 管理员
actor User as 普通用户
rectangle 仪表盘 {
  Admin -- (查看房态与统计)
  Admin -- (快捷办理入住)
  Admin -- (快捷退房)
  Admin -- (指派打扫)
  Admin -- (结束维修)
  User -- (查看运行概览)
}
```

### 人员管理
- 新增：填写姓名与岗位，提交后员工即刻入库。
- 编辑：在表格中点击“编辑”可更新姓名和岗位。
- 删除：点击“删除”移除员工，系统会把相关打扫记录的人员字段置空。

```mermaid
usecaseDiagram
actor Admin as 管理员
rectangle 人员管理 {
  Admin -- (新增员工)
  Admin -- (编辑员工信息)
  Admin -- (删除员工)
}
```

### 房间管理
- 新增：输入房间号、楼层、容量即可创建，可在仪表盘卡片上处理入住/退房/打扫/维修。
- 编辑：更新房间号、楼层或容量。
- 删除：移除房间的同时删除其预订与打扫记录。
- 房间状态：
  - **可用**：可办理入住；
  - **入住**：可退房并指派打扫；
  - **打扫中**：完成打扫后可标记为“已打扫”；
  - **维修中**：可切换为“结束维修”。

```mermaid
usecaseDiagram
actor Admin as 管理员
actor User as 普通用户
rectangle 房间管理 {
  Admin -- (新增房间)
  Admin -- (编辑房间信息)
  Admin -- (删除房间)
  Admin -- (维护房态)
  User -- (查看房间列表)
}
```

### 房间打扫
- 创建任务：选择房间和保洁人员生成打扫记录，房间状态自动变为“打扫中”。
- 进行中列表：展示未完成任务，可点击“标记完成”恢复房间为可用。
- 历史记录：查看最近完成的打扫记录。

```mermaid
usecaseDiagram
actor Admin as 管理员
actor Cleaner as 保洁员
rectangle 房间打扫 {
  Admin -- (创建打扫任务)
  Admin -- (查看进行中任务)
  Admin -- (查看历史记录)
  Cleaner -- (更新打扫完成)
}
```

### 布草管理
- 新增：填写名称、单价与状态（库存/使用中/清洗中）。
- 统计：模块顶部显示布草总量以及各状态占比。
- 编辑/删除：在表格行内即可修改或移除布草记录。

```mermaid
usecaseDiagram
actor Admin as 管理员
rectangle 布草管理 {
  Admin -- (新增布草)
  Admin -- (编辑布草状态与价格)
  Admin -- (删除布草记录)
  Admin -- (查看布草统计)
}
```

### 资产管理
- 新增：填写资产名称、类别、位置、购入日期与估值。
- 查看：表格展示资产清单（默认显示最新 50 条）。
- 编辑/删除：在列表行内对资产进行修改或删除。

```mermaid
usecaseDiagram
actor Admin as 管理员
rectangle 资产管理 {
  Admin -- (新增资产)
  Admin -- (查看资产清单)
  Admin -- (编辑资产信息)
  Admin -- (删除资产记录)
}
```

### 用户管理
- 新增：录入用户名并选择角色（管理员/普通用户），登录密码统一为默认值。
- 编辑：更新用户名或角色，若修改当前登录用户会同步刷新登录状态。
- 删除：移除用户，若删除的是当前用户会自动退出登录。

```mermaid
usecaseDiagram
actor Admin as 管理员
rectangle 用户管理 {
  Admin -- (新增用户)
  Admin -- (编辑用户角色)
  Admin -- (删除用户)
}
```

### 系统管理
- 重置示例数据：恢复初始化的演示数据集。
- 导出当前数据：将 `localStorage` 中的业务数据以 JSON 文件下载。
- 清空数据：删除除用户以外的所有业务数据，便于重新演示。

```mermaid
usecaseDiagram
actor Admin as 管理员
rectangle 系统管理 {
  Admin -- (重置示例数据)
  Admin -- (导出业务数据)
  Admin -- (清空业务数据)
}
```

## 数据持久化与注意事项
- 所有操作即时保存到浏览器本地存储；切换页面或刷新浏览器后数据仍保留。
- 如需重新开始演示，可使用“系统管理”中的“重置示例数据”或“清空数据”。
- 导出文件可用作备份或迁移到其他浏览器（需手动导入到 `localStorage`）。
